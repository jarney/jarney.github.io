315 // We want a 'minimal' mir to represent what's
316 // going on here.
317 // Scope {
318 // std::vector<ScopeOperation>
319 // Scope *parent;
320 // std::vector<Gyoji::owned<Scope>> children;
321 // };
322 //
323 // This tells us what's going on in each scope,
324 // but skips things like operations that don't impact
325 // branches, just deals with 'abstractions' of scopes.
326 // An 'if' statement, for example, will simply introduce
327 // two child scopes.
328 // ScopeOperation {
329 // enum { VARDECL, LABEL, GOTO, CHILDREN}
330 // };
331 // Then once we've extracted the scopes,
332 // we can 'dump' the whole thing and analyze whether
333 // the jumps are legal according to the rules because
334 // we have all the information we need. We can look
335 // at each goto and find the corresponding
336 // labels and emit errors. All of this should
337 // happen during the first pass before the MIR
338 // is generated because we'll need all this info
339 // to generate a valid MIR.
340 // Perhaps this can live in the 'type-resolver' although
341 // it's not really related to types.
342 //
343 // With this, we can simply analyze for each goto,
344 // whether there is a corresponding label defined somewhere
345 // and check that it's legal in terms of scope
346 // rules. If it's not defined anywhere, that's an error.
347 // If it's defined and skips an initialization, that's also
348 // an error. We also have enough information in the goto
349 // itself to figure out what to unwind to get to the target.
350
351 // At the top-level, we'll want a map of label to scope
352 // so that when we do our analysis, we can find the label
353 // corresponding to a goto.
