Detailed Description

This represents an if condition in one of several forms.

The if/else construct in JLang comes in several forms. It is designed slightly differently than the C equivalent in order to ensure that it does not have the "dangling else" ambiguity. The curly-braces are mandatory as a part of eliminating that ambiguity. The only excption to this case is the 'else if' construct which does NOT require curly braces after the 'else' and allows very familiar semantics.

The following are examples of the various forms that the if construct may take.

If without else: 

...
if (foo) {
   bar();
}
...

If with else. Note that in this case, the curly braces '{' are mandatory for both the if and else blocks. 

...
if (foo) {
   bar();
}
else {
   baz();
}
...

If/else if/else if chain. Note that ONLY in this case, we are allowed to skip the curly brace between the 'else' and the next 'if', allowing the semantics to be unambiguious.

...
if (foo) {
   bar();
}
else if (q) {
   baz();
}
else {
   end();
}
...
